内置数学 SQL 函数

1.概述

下面显示的数学函数是标量函数的子组，这些函数内置于 SQLite 合并源文件中，但仅处于活动状态 如果使用 -DSQLITE_ENABLE_MATH_FUNCTIONS compile-time 选项编译合并。

数学函数的参数可以是整数、浮点数、 或者看起来像整数或实数的字符串或 blob。如果有任何参数 为 NULL 或不容易转换为数字的字符串或 blob，则 则函数将返回 NULL。 这些函数还会为域错误返回 NULL，例如尝试 取负数的平方根，或计算 a 的反余弦 值大于 1.0 或小于 -1.0。

这些函数返回的值通常是近似值。 例如，pi（） 函数返回 3.141592653589793115997963468544185161590576171875 其中 大约是 1.22465E-16 太小了，但它是最接近的近似值 IEEE754双打。

2. 内置标量 SQL 数学函数的描述

acos(X)

返回 X 的反余弦。结果以弧度为单位。

acosh(X)

返回 X 的双曲反余弦。

asin(X)

返回 X 的反正弦值。结果以弧度为单位。

asinh(X)

返回 X 的双曲反正弦。

atan(X)

返回 X 的反正切值。结果以弧度为单位。

atan2(Y,X)

返回 Y/X 的反正切值。结果以弧度为单位。这 根据符号将结果放入正确的象限 X 和 Y 的 X 和 Y 的 X 和 Y 的 X 和

atanh(X)

返回 X 的双曲反正切值。

ceil(X)
ceiling(X)

返回第一个大于或等于 X 的可表示整数值。 对于 X 的正值，此例程从零开始舍入。 对于 X 的负值，此例程将四舍五入为零。

cos(X)

返回 X 的余弦值。X 以弧度为单位。

cosh(X)

返回 X 的双曲余弦值。

degrees(X)

将值 X 从弧度转换为度数。

exp(X)

计算 e（欧拉数，约为 2.71828182845905）引发 到幂 X。

floor(X)

返回第一个小于或等于 X 的可表示整数值。 对于正数，此函数将四舍五入为零。 对于负数，此函数从零开始四舍五入。

ln(X)

返回 X 的自然对数。

log(X)
log10(X)
log(B,X)

返回 X 的以 10 为底的对数。或者，对于双参数版本， 返回 X 的以 B 为底的对数。

兼容性说明：SQLite 的工作方式与 PostgreSQL 类似，因为 log（） 函数 计算以 10 为基数的对数。大多数其他 SQL 数据库引擎计算 log（） 的自然对数。在 log（B，X） 的双参数版本中， 第一个参数是基数，第二个参数是作数。这是 与 PostgreSQL 和 MySQL 中的相同，但与 SQL Server 相反，后者 使用第二个参数作为基，使用第一个参数作为作数。

log2(X)

返回数字 X 的对数 base-2。

mod(X,Y)

返回 X 除以 Y 后的余数。这类似于 '%' 运算符，但它适用于非整数参数。

pi()

Return an approximation for π.

pow(X,Y)
power(X,Y)

计算 X 的 Y 次幂。

radians(X)

将 X 从度数转换为弧度。

sin(X)

返回 X 的正弦值。X 以弧度为单位。

sinh(X)

返回 X 的双曲正弦值。

sqrt(X)

返回 X 的平方根，如果 X 为负数，则返回 NULL。

tan(X)

返回 X 的正切值。X 以弧度为单位。

tanh(X)

返回 X 的双曲正切值。

trunc(X)

返回介于 X 和 0 之间（包括 X 和 0）的可表示整数 那离零最远。或者，换句话说，返回 X 的整数部分，四舍五入为零。 trunc（） 函数类似于 ceiling（X） 和 floor（X），不同之处在于 它总是向零舍入，而 ceiling（X） 和 floor（X） 舍入 分别是 up 和 down。
