DROP TABLE

drop-table-stmt 中：

DROP TABLE 语句将删除使用 CREATE TABLE 语句添加的表。指定的名称是 table name 的已删除的表将从数据库中完全删除 schema 和磁盘文件。无法恢复 table。 所有索引和触发器 与该表关联的数据也会被删除。

可选的 IF EXISTS 子句会抑制通常会 result（如果表不存在）。

如果启用了外键约束，则 DROP TABLE 命令将执行 隐式 DELETE FROM 命令，然后再删除 表。附加到表的任何触发器都是 在隐式 DELETE FROM 之前从数据库架构中删除 执行，因此这不会导致任何触发器触发。相比之下，一个 隐式 DELETE FROM 确实会导致发生任何配置的外键作。 如果执行了隐式 DELETE FROM 作为 DROP TABLE 命令的一部分违反了任何直接外键约束， 将返回错误，并且不会删除该表。如果 隐式 DELETE FROM 会导致任何 defered 外键约束，并且冲突仍然 存在，则在提交事务时返回错误 的提交。
