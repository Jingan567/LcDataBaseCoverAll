SQLite Keywords

SQL 标准指定了大量关键字，这些关键字可能不是 用作表、索引、列、数据库、用户定义的名称 函数、排序规则、虚拟表模块或任何其他命名对象。 关键字列表很长，很少有人能记住它们。 对于大多数 SQL 代码，最安全的做法是永远不要使用任何英语 word 作为用户定义对象的名称。

如果你想用一个关键词作为名字，你需要引用它。那里 是在 SQLite 中引用关键字的四种方式：

'关键字' 单引号中的关键字是字符串文本。
“关键字” 双引号中的关键字是标识符。
[关键词] 括在方括号中的关键字是 标识符。这不是标准 SQL。这个引用机制 由 MS Access 和 SQL Server 使用，并包含在 SQLite 中 兼容性。
'关键字' 用重音符括起来的关键字（ASCII 代码 96）是 标识符。这不是标准 SQL。这个引用机制 由 MySQL 使用，并包含在 SQLite 中 兼容性。

为了在面对历史 SQL 语句时具有弹性，SQLite 有时会扭曲上述引用规则：

  • 如果单个 引号（例如： 'key' 或 'glob'）用于 允许使用标识符，但如果不允许使用字符串文字，则 令牌被理解为标识符而不是字符串文本。

  • 如果 double 中的关键字 引号（例如：“key”或“glob”）用于 它不能解析为标识符，但其中字符串文本 允许，则令牌将理解为字符串文本 的标识符。

提醒程序员不要使用 前面的项目符号。我们强调，它们的存在只是为了让 格式不正确的 SQL 语句将正确运行。的未来版本 SQLite 可能会引发错误，而不是接受格式错误的 上述例外情况涵盖的 statements。

SQLite 在具有新功能时会不时添加新关键字。 因此，为了防止您的代码被未来的增强功能破坏，您应该 通常引用任何英语单词的标识符，即使 你不必这样做。
