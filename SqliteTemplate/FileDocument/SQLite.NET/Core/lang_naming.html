在 SQLite 中，标识数据库对象（表、索引、触发器或视图） 通过对象的名称和它所在的数据库的名称。 数据库对象可能驻留在主数据库、临时数据库或 附加的数据库。

DROP TABLE、DROP INDEX、DROP VIEW、DROP TRIGGER、REINDEX、ALTER TABLE 和许多其他命令的语法都允许用户 指定数据库对象，可以单独使用其名称，也可以指定 其名称及其数据库的名称。如果未将数据库指定为 part 对象引用中，则 SQLite 会搜索 main、temp 和所有附加的 databases 查找具有匹配名称的对象。搜索临时数据库 首先是 main 数据库，然后是 命令他们被附上。引用解析为第一个匹配项 发现。例如：

      /* Add a table named 't1' to the temp, main and an attached database */
      ATTACH 'file.db' AS aux;
      CREATE TABLE t1(x, y);
      CREATE TEMP TABLE t1(x, y);
      CREATE TABLE aux.t1(x, y);

      DROP TABLE t1;         /* Drop table in temp database */
      DROP TABLE t1;         /* Drop table in main database */
      DROP TABLE t1;         /* Drop table in aux database */

如果将架构名称指定为对象引用的一部分，则它必须为 “main”或“temp”或附加数据库的架构名称。 与其他 SQL 标识符一样，架构名称不区分大小写。 如果指定了 schema name，则只搜索该 a schema 命名对象。

大多数对象引用只能解析为特定类型的对象（对于 示例，作为 DROP TABLE 语句一部分的引用只能解析 传递给 table 对象，而不是索引、触发器或视图）。但是，在某些情况下 （例如 REINDEX）一个对象引用可以解析为多个类型 的对象。在数据库架构中搜索命名对象时， 不能在引用上下文中使用的类型始终是 忽视。
