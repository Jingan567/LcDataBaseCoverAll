重新索引

reindex-stmt 中：

REINDEX 命令用于从头开始删除和重新创建索引。 当排序规则序列的定义发生更改时，这很有用，或者 当表达式上的索引涉及其定义的函数时 已更改。

如果 REINDEX 关键字后面没有排序规则序列或数据库 Object Identifier，则重新构建所有附加数据库中的所有索引。

如果 REINDEX 关键字后跟排序规则序列名称，则 使用命名排序规则序列的所有附加数据库中的所有索引 重新创建。

或者，如果附加到 REINDEX 的参数标识了特定的 database 表，则重新构建附加到 database 表的所有索引。 如果它标识了特定的数据库索引，则仅重新创建该索引。

对于“REINDEX name”形式的命令，匹配 against collation-name 优先于匹配项 针对 index-name 或 table-name。 在重新索引特定表或索引时，可以通过始终指定 schema-name 来避免语法中的这种歧义。
