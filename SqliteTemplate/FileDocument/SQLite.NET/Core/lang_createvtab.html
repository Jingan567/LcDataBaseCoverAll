创建虚拟表

create-virtual-table-stmt 中：

虚拟表是外部存储或计算的接口 引擎，该引擎似乎是一个表，但实际上并不存储信息 在数据库文件中。

通常，您可以使用虚拟表执行任何可以完成的作 替换为普通表，但不能在 虚拟表。某些虚拟表实现可能会施加额外的 限制。例如，许多虚拟表是只读的。

module-name 是实现 虚拟表。module-name 必须注册到 在发出 CREATE VIRTUAL TABLE 语句之前使用 sqlite3_create_module（） 或 sqlite3_create_module_v2（） 的 SQLite 数据库连接。 该模块采用零个或多个逗号分隔的参数。 参数可以是几乎任何文本，只要它保持平衡 括弧。参数语法足够通用，以至于 可以将参数设置为在传统 CREATE TABLE 语句中显示为列定义。 SQLite 直接传递模块参数 到模块实现的 xCreate 和 xConnect 方法 没有任何解释。这是责任 来解析和解释自己的参数。

使用普通的 DROP TABLE 语句销毁虚拟表。没有 DROP VIRTUAL TABLE 语句。
