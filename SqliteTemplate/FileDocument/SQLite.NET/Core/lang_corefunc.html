3. 内置标量 SQL 函数说明

绝对 （X）

abs（X） 函数返回数字 如果 X 为 NULL，则 Abs（X） 返回 NULL。 如果 X 是字符串或 blob，则 Abs（X） 返回 0.0 不能转换为数值。如果 X 是 integer -9223372036854775808 然后 abs（X） 引发整数溢出 错误，因为没有等效的正 64 位 2 补码值。

更改（）

changes（） 函数返回已更改的数据库行数 或者由最近完成的 INSERT、DELETE、 或 UPDATE 语句，不包括较低级别触发器中的语句。 changes（） SQL 函数是 sqlite3_changes64（） C/C++ 函数的包装器，因此遵循相同的规则来计算更改。

字符 （X1，X2,...,XN）

char（X1，X2,...,XN） 函数返回一个字符串，该字符串由具有 分别是整数 X1 到 XN 的 unicode 码位值。

coalesce（X，Y,...）

coalesce（） 函数返回其第一个非 NULL 参数的副本，或者 如果所有参数均为 NULL，则为 NULL。Coalesce（） 必须至少具有 2 个参数。

concat（X,...）

concat（...） 函数返回一个字符串，该字符串是 string 表示形式。如果所有参数都是 NULL，则 concat（） 返回一个空字符串。

concat_ws（SEP，X,...）

concat_ws （SEP,...） 函数返回一个字符串，该字符串是 第一个参数之外的所有非 null 参数，使用 first 参数作为分隔符。如果第一个参数为 NULL，则 concat_ws（） 返回 NULL。如果除第一个参数以外的所有参数均为 NULL，则 concat_ws（） 返回一个空字符串。

format（格式,...）

格式 （FORMAT,...）SQL 函数的工作方式类似于 sqlite3_mprintf（） C 语言 函数和 printf（） 函数。 第一个参数是一个格式字符串，用于指定如何构造输出 string 使用从后续参数中获取的值。如果 FORMAT 参数为 missing 或 NULL 则结果为 NULL。%n 格式将被静默忽略，并且 不使用参数。%p 格式是 %X 的别名。%z 格式 可与 %s 互换。如果参数列表中的参数太少， 假定缺少的参数具有 NULL 值，该值将转换为 数字格式为 0 或 0.0，%s 为空字符串。有关更多信息，请参阅内置的 printf（） 文档。

glob（X，Y）

glob（X，Y） 函数等效于 表达式 “Y GLOB X”。 请注意，X 和 Y 参数在 glob（） 函数中是相反的 相对于中缀 GLOB 运算符。Y 是字符串，X 是 模式。因此，例如，以下表达式是等效的：

     name GLOB '*helium*'
     glob('*helium*',name)

如果使用 sqlite3_create_function（） 接口 然后用替代实现覆盖 glob（X，Y） 函数 GLOB 运算符将调用替代实现。

十六进制 （X）

hex（） 函数将其参数解释为 BLOB 并返回 一个字符串，它是 那个 blob。

如果 “hex（X）” 中的参数 X 是 整数或浮点数，则“将其参数解释为 BLOB”意味着 首先将二进制数转换为 UTF8 文本表示形式，然后 该文本被解释为 BLOB。因此，“hex（12345678）” 渲染 作为 “3132333435363738” 而不是整数值的二进制表示形式 “0000000000BC614E”。

参见：unhex（）

如果为空 （X，Y）

ifnull（） 函数返回其第一个非 NULL 参数的副本，或者 如果两个参数均为 NULL，则为 NULL。Ifnull（） 必须恰好有 2 个参数。 ifnull（） 函数等效于具有两个参数的 coalesce（）。

iif（X，Y，Z）

如果 X 为 true，则 iif（X，Y，Z） 函数返回值 Y，否则返回 Z。 iif（X，Y，Z） 函数在逻辑上等同于 CASE 表达式 “CASE WHEN X THEN Y ELSE Z END” 并生成与 CASE 表达式相同的字节码。

instr（X，Y）

instr（X，Y） 函数在 string X 并返回前面的字符数加 1，如果返回 0，则返回 0 在 X 中找不到 Y。 或者，如果 X 和 Y 都是 BLOB，则 instr（X，Y） 返回 1 大于第一次出现 Y 之前的字节数，如果为 0，则为 0 Y 不会出现在 X 轴内的任何位置。 如果 instr（X，Y） 的参数 X 和 Y 都是非 NULL 且不是 BLOB 然后两者都被解释为字符串。 如果 X 或 Y 在 instr（X，Y） 中为 NULL，则结果为 NULL。

last_insert_rowid（）

last_insert_rowid（） 函数从调用了 功能。 last_insert_rowid（） SQL 函数是 sqlite3_last_insert_rowid（） C/C++ 接口函数的包装器。

长度 （X）

对于字符串值 X，length（X） 函数返回 字符（不是字节）在 X 中的第一个 NUL 字符之前。 由于 SQLite 字符串通常不包含 NUL 字符，因此 length（X） function 通常会返回字符串 X 中的字符总数。 对于 blob 值 X，length（X） 返回 blob 中的字节数。 如果 X 为 NULL，则 length（X） 为 NULL。 如果 X 是数字，则 length（X） 返回字符串的长度 X 的表示。

请注意，对于字符串，length（X） 函数返回字符串的字符长度，而不是字节长度。字符长度是数字 字符串中的字符数。字符长度始终不同于 UTF-16 字符串的字节长度，并且可以与字节长度不同 对于 UTF-8 字符串，如果字符串包含多字节字符。使用 octet_length（） 函数查找字符串的字节长度。

对于 BLOB 值，length（X） 始终返回 BLOB 的字节长度。

对于字符串值，length（X） 必须按顺序将整个字符串读入内存 以计算字符长度。但对于 BLOB 值，这不是必需的，因为 SQLite 知道 BLOB 中有多少字节。因此，对于多兆字节的值， length（X） 函数通常比字符串快得多，因为 它不需要将值加载到内存中。

like（X，Y）
like（X，Y，Z）

like（） 函数用于实现 “Y LIKE X [ESCAPE Z]”表达式。 如果存在可选的 ESCAPE 子句，则 like（） 函数使用三个参数调用。否则，它是 仅使用两个参数调用。请注意，X 和 Y 参数为 在 like（） 函数中相对于中缀 LIKE 运算符相反。 X 是模式， Y 是要与该模式匹配的字符串。 因此，以下表达式是等效的：

     name LIKE '%neon%'
     like('%neon%',name)

sqlite3_create_function（） 接口可用于覆盖 like（） 函数，从而更改 LIKE 运算符的作。当覆盖 like（） 函数时，它可能很重要 覆盖 like（） 的两个和三个参数版本 功能。否则，可能会调用不同的代码来实现 LIKE 运算符，具体取决于 ESCAPE 子句是否为 指定。

似然 （X，Y）

likelihood（X，Y） 函数返回参数 X 不变。 likelihood（X，Y） 中的值 Y 必须是浮点常数 介于 0.0 和 1.0 之间（包括 0.0 和 1.0）。 likelihood（X） 函数是代码生成器的 no-op 优化，使其在运行时不消耗 CPU 周期 （即，在调用 sqlite3_step（） 期间）。 likelihood（X，Y） 函数的目的是提供提示 对于查询计划者来说，参数 X 是一个布尔值，即 true，概率约为 Y。 unlikely（X） 函数是 likelihood（X，0.0625） 的简写。 likely（X） 函数是 likelihood（X，0.9375） 的简写。

可能 （X）

likely（X） 函数返回参数 X 不变。 likely（X） 函数是代码生成器 优化 out，使其在 运行时（即，在调用 sqlite3_step（） 期间）。 likely（X） 函数的目的是提供提示 对于查询计划者，参数 X 是布尔值 这通常是正确的。likely（X） 函数是等效的 到 likelihood（X，0.9375）。参见：unlikely（X）。

load_extension（X）
load_extension（X，Y）

load_extension（X，Y） 函数从共享的 使用入口点 Y 的名为 X 的库文件。load_extension（） 的结果 始终为 NULL。如果省略 Y，则使用默认入口点名称。 load_extension如果扩展无法 正确加载或初始化。

如果扩展尝试 load_extension（） 函数 修改或删除 SQL 函数或排序序列。这 extension 可以添加新函数或整理序列，但不能 修改或删除现有函数或整理序列，因为 这些函数和/或整理序列可能在其他地方使用 在当前运行的 SQL 语句中。要加载一个扩展，请 更改或删除函数或整理序列，请使用 sqlite3_load_extension（） C 语言 API。

出于安全原因，默认情况下禁用扩展加载，并且必须 通过事先调用 sqlite3_enable_load_extension（） 来启用。

下 （X）

lower（X） 函数返回包含所有 ASCII 字符的字符串 X 的副本 转换为小写。默认的内置 lower（） 函数有效 仅适用于 ASCII 字符。对非 ASCII 执行大小写转换 字符，加载 ICU 扩展。

ltrim（X）
ltrim（X，Y）

ltrim（X，Y） 函数返回通过删除 any 和 all 而形成的字符串 从 X 的左侧出现在 Y 中的字符。 如果省略 Y 参数，则 ltrim（X） 会删除左侧的空格 的 X。

最大 （X，Y,...）

多参数 max（） 函数返回带有 maximum 值，如果任何参数为 NULL，则返回 NULL。 多参数 max（） 函数从左到右搜索其参数 对于定义 collating 函数并使用该 collating 的参数 函数进行所有字符串比较。如果 max（） 的参数都没有 定义一个 collating 函数，则使用 BINARY collating 函数。 请注意，max（） 是一个简单的函数，当 它有 2 个或更多参数，但如果只给出一个参数，则作为聚合函数运行。

min（X，Y,...）

多参数 min（） 函数返回带有 最小值。 多参数 min（） 函数从左到右搜索其参数 对于定义 collating 函数并使用该 collating 的参数 函数进行所有字符串比较。如果 min（） 的参数都没有 定义一个 collating 函数，则使用 BINARY collating 函数。 请注意，min（） 是一个简单的函数，当 它有 2 个或更多参数，但如果给定，则作为聚合函数运行 只有一个参数。

空if（X，Y）

如果参数为 different 和 NULL 如果参数相同。nullif（X，Y） 函数 从左到右搜索其参数，以查找定义 collating 函数，并将该 collating 函数用于所有字符串 比较。如果 nullif（） 的参数均未定义整理函数 然后使用 BINARY 整理函数。

octet_length（X）

octet_length（X） 函数返回编码中的字节数 的文本字符串 X。如果 X 为 NULL，则 octet_length（X） 返回 NULL。如果 X 为 一个 BLOB 值，则 octet_length（X） 与 length（X） 相同。如果 X 是 numeric 值，则 octet_length（X） 返回文本中的字节数 该数字的渲染。

因为 octet_length（X） 返回的是 X 中的字节数，而不是数字 的字符中，返回的值取决于数据库编码。这 octet_length（） 函数可以为同一输入字符串返回不同的答案 如果数据库编码为 UTF16 而不是 UTF8。

如果参数 X 是表列，并且值是 text 或 blob 类型， 则 octet_length（X） 避免从磁盘读取 X 的内容，因为字节 length 可以从元数据中计算。因此，octet_length（X） 是有效的 即使 X 是包含数兆文本或 blob 值的列。

printf（格式,...）

printf（） SQL 函数是 format（） SQL 函数的别名。 format（） SQL 函数最初命名为 printf（）。但这个名字是 后来更改为 format（） 以与其他数据库引擎兼容。 printf（） 名称保留为别名，以免破坏遗留代码。

quote（X）

quote（X） 函数返回 SQL 文本的文本，该文本 是适合包含在 SQL 语句中的参数的值。 字符串用单引号括起来，内部引号上有转义符 根据需要。BLOB 编码为十六进制文本。 嵌入了 NUL 字符的字符串不能表示为字符串 Literals 的 Literals 中，因此返回的字符串 Literals 之前被截断 到第一个 NUL。

unicode（X） 函数返回对应于 字符串 X 的第一个字符。如果 unicode（X） 的参数不是字符串 则结果是 undefined。

不可能 （X）

unlikely（X） 函数返回参数 X 不变。 unlikely（X） 函数是代码生成器 优化 out，使其在 运行时（即，在调用 sqlite3_step（） 期间）。 unlikely（X） 函数的目的是提供提示 对于查询计划者，参数 X 是布尔值 这通常不是真的。unlikely（X） 函数是等效的 到 likelihood（X， 0.0625）。

上部 （X）

upper（X） 函数返回输入字符串 X 的副本，其中所有 小写 ASCII 字符将转换为其大写等效字符。

零斑点 （N）

zeroblob（N） 函数返回一个由 N 字节的 0x00 组成的 BLOB。 SQLite 非常有效地管理这些 zeroblob。Zeroblobs 可用于 为稍后使用增量 BLOB I/O 写入的 BLOB 预留空间。 此 SQL 函数是使用 C/C++ 接口中的 sqlite3_result_zeroblob（） 例程实现的。
