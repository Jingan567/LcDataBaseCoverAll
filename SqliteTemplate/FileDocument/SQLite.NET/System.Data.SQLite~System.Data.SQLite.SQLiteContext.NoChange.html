SQLiteContext.NoChange Method

Attempts to determine if a specific column being fetched via the xColumn method as part of an UPDATE operation will not change. Use of this method is invalid in any other context.

public int NoChange();

Return Value

Non-zero if the column being fetched via a pending call to the xColumn method as part of an UPDATE operation during which the column value will not change. The virtual table implementation can use this hint as permission to substitute a return value that is less expensive to compute and that the corresponding xUpdate method understands as a "no-change" value.
