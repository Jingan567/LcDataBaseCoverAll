ISQLiteSession 方法

下面列出了 ISQLiteSession 接口的方法。有关 ISQLiteSession 接口成员的完整列表，请参阅 ISQLiteSession 成员主题。

公共实例方法

AttachTable 成功后，将开始跟踪对指定表的更改。任何受此方法调用影响的表都不会导致调用 SessionTableFilterCallback 回调。
CreateChangeSet重载。尝试通过 rawData 创建并返回此 session 实例表示的组合更改集。
CreatePatchSet重载。尝试通过 rawData 创建并返回此会话实例表示为补丁集的组合更改集。
GetMemoryBytesInUse 此方法尝试确定会话使用的内存量。
IsEmpty 确定此会话的数据中当前是否有任何修订。
IsEnabled 确定此会话当前是否正在跟踪对其关联数据库的更改。
IsIndirect 确定此会话当前是否设置为将更改标记为间接更改 （即，就像更改是通过触发器或外键作进行的） 一样。
LoadDifferencesFromTable 此方法将两个 table 之间的差异 [具有相同的名称、列集和主键定义] 加载到此 session 实例中。
SetTableFilter该方法用于设置该实例的表筛选条件。
SetToDirect 清除此会话的间接标志。后续更改将被标记为 direct，直到再次更改此标志。
SetToDisabled 禁用对关联数据库的更改跟踪。
SetToEnabled 启用对关联数据库的更改跟踪。
SetToIndirect 设置此会话的间接标志。后续更改将被标记为 间接 ，直到再次更改此标志。
