SQLiteConfigDbOpsEnum Enumeration

These are the supported configuration verbs for use with the native SQLite library. They are used with the SetConfigurationOption method.

Member Name Description
SQLITE_DBCONFIG_NONE This value represents an unknown (or invalid) option, do not use it.
SQLITE_DBCONFIG_MAINDBNAME This option is used to change the name of the "main" database schema. The sole argument is a pointer to a constant UTF8 string which will become the new schema name in place of "main".
SQLITE_DBCONFIG_LOOKASIDE This option is used to configure the lookaside memory allocator. The value must be an array with three elements. The second element must be an Int32 containing the size of each buffer slot. The third element must be an Int32 containing the number of slots. The first element must be an IntPtr that points to a native memory buffer of bytes equal to or greater than the product of the second and third element values.
SQLITE_DBCONFIG_ENABLE_FKEY This option is used to enable or disable the enforcement of foreign key constraints.
SQLITE_DBCONFIG_ENABLE_TRIGGER This option is used to enable or disable triggers.
SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER This option is used to enable or disable the two-argument version of the fts3_tokenizer() function which is part of the FTS3 full-text search engine extension.
SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION This option is used to enable or disable the loading of extensions.
SQLITE_DBCONFIG_NO_CKPT_ON_CLOSE This option is used to enable or disable the automatic checkpointing when a WAL database is closed.
SQLITE_DBCONFIG_ENABLE_QPSG This option is used to enable or disable the query planner stability guarantee (QPSG).
SQLITE_DBCONFIG_TRIGGER_EQP This option is used to enable or disable the extra EXPLAIN QUERY PLAN output for trigger programs.
SQLITE_DBCONFIG_RESET_DATABASE This option is used as part of the process to reset a database back to an empty state. Because resetting a database is destructive and irreversible, the process requires the use of this obscure flag and multiple steps to help ensure that it does not happen by accident.
SQLITE_DBCONFIG_DEFENSIVE This option activates or deactivates the "defensive" flag for a database connection. When the defensive flag is enabled, language features that allow ordinary SQL to deliberately corrupt the database file are disabled. The disabled features include but are not limited to the following: <ul><li> The PRAGMA writable_schema=ON statement. </li><li> The PRAGMA journal_mode=OFF statement. </li><li> Writes to the sqlite_dbpage virtual table. </li><li> Direct writes to shadow tables. </li></ul>
SQLITE_DBCONFIG_WRITABLE_SCHEMA This option activates or deactivates the "writable_schema" flag.
SQLITE_DBCONFIG_LEGACY_ALTER_TABLE This option activates or deactivates the legacy behavior of the ALTER TABLE RENAME command such it behaves as it did prior to version 3.24.0 (2018-06-04).
SQLITE_DBCONFIG_DQS_DML This option activates or deactivates the legacy double-quoted string literal misfeature for DML statement only, that is DELETE, INSERT, SELECT, and UPDATE statements.
SQLITE_DBCONFIG_DQS_DDL This option activates or deactivates the legacy double-quoted string literal misfeature for DDL statements, such as CREATE TABLE and CREATE INDEX.
SQLITE_DBCONFIG_ENABLE_VIEW This option is used to enable or disable CREATE VIEW.
SQLITE_DBCONFIG_LEGACY_FILE_FORMAT This option activates or deactivates the legacy file format flag.
SQLITE_DBCONFIG_TRUSTED_SCHEMA This option tells SQLite to assume that database schemas (i.e. the contents of the sqlite_master tables) are untainted by malicious content. When the trusted schema option is disabled, SQLite takes additional defensive steps to protect the application from harm including: <ul><li> Prohibit the use of SQL functions inside triggers, views, CHECK constraints, DEFAULT clauses, expression indexes, partial indexes, or generated columns unless those functions are tagged with SQLITE_INNOCUOUS. </li><li> Prohibit the use of virtual tables inside of triggers or views unless those virtual tables are tagged with SQLITE_VTAB_INNOCUOUS. </ul> This setting defaults to "on" for legacy compatibility, however all applications are advised to turn it off if possible. This setting can also be controlled using the PRAGMA trusted_schema statement.
SQLITE_DBCONFIG_STMT_SCANSTATUS This option is only useful in SQLITE_ENABLE_STMT_SCANSTATUS builds. In this case, it sets or clears a flag that enables collection of the sqlite3_stmt_scanstatus_v2() statistics. For statistics to be collected, the flag must be set on the database handle both when the SQL statement is prepared and when it is stepped. The flag is set (collection of statistics is enabled) by default.
SQLITE_DBCONFIG_REVERSE_SCANORDER The SQLITE_DBCONFIG_REVERSE_SCANORDER option change the default order in which tables and indexes are scanned so that the scans start at the end and work toward the beginning rather than starting at the beginning and working toward the end. Setting SQLITE_DBCONFIG_REVERSE_SCANORDER is the same as setting [PRAGMA reverse_unordered_selects]. This configuration option is useful for application testing.
