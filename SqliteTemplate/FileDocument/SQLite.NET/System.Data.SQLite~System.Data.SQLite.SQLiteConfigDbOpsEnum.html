SQLiteConfigDbOpsEnum 枚举

这些是支持用于本机 SQLite 库的配置动词。它们与 SetConfigurationOption 方法一起使用。

public enum SQLiteConfigDbOpsEnum

Members

Member Name Description
SQLITE_DBCONFIG_NONE 此值表示未知（或无效）选项，请勿使用它。
SQLITE_DBCONFIG_MAINDBNAME 此选项用于更改 “main” 数据库架构的名称。唯一的参数是指向常量 UTF8 字符串的指针，该字符串将成为新的架构名称，而不是 “main”。
SQLITE_DBCONFIG_LOOKASIDE 此选项用于配置后备内存分配器。该值必须是包含三个元素的数组。第二个元素必须是包含每个缓冲区槽大小的 Int32。第三个元素必须是包含槽数的 Int32。第一个元素必须是指向字节等于或大于第二个和第三个元素值的乘积的字节的本机内存缓冲区的 IntPtr。
SQLITE_DBCONFIG_ENABLE_FKEY 此选项用于启用或禁用外键约束的实施。
SQLITE_DBCONFIG_ENABLE_TRIGGER 此选项用于启用或禁用触发器。
SQLITE_DBCONFIG_ENABLE_FTS3_TOKENIZER 此选项用于启用或禁用fts3_tokenizer（）函数的双参数版本，该函数是FTS3全文搜索引擎扩展的一部分。
SQLITE_DBCONFIG_ENABLE_LOAD_EXTENSION 此选项用于启用或禁用扩展的加载。
SQLITE_DBCONFIG_NO_CKPT_ON_CLOSE 此选项用于在关闭 WAL 数据库时启用或禁用自动检查点。
SQLITE_DBCONFIG_ENABLE_QPSG 此选项用于启用或禁用查询计划程序稳定性保证 （QPSG）。
SQLITE_DBCONFIG_TRIGGER_EQP 此选项用于启用或禁用触发器程序的额外 EXPLAIN QUERY PLAN 输出。
SQLITE_DBCONFIG_RESET_DATABASE 此选项用作将数据库重置回空状态的过程的一部分。由于重置数据库具有破坏性且不可逆，因此该过程需要使用此模糊标志和多个步骤来帮助确保它不会意外发生。
SQLITE_DBCONFIG_DEFENSIVE 此选项可激活或停用数据库连接的 “defensive” 标志。启用 defensive 标志后，将禁用允许普通 SQL 故意破坏数据库文件的语言功能。禁用的功能包括但不限于以下内容： <ul><li> The PRAGMA writable_schema=ON statement. </li><li> The PRAGMA journal_mode=OFF statement. </li><li> Writes to the sqlite_dbpage virtual table. </li><li> Direct writes to shadow tables. </li></ul>
SQLITE_DBCONFIG_WRITABLE_SCHEMA 此选项可激活或停用 “writable_schema” 标志。
SQLITE_DBCONFIG_LEGACY_ALTER_TABLE 此选项可激活或停用 ALTER TABLE RENAME 命令的旧行为，使其行为与版本 3.24.0 （2018-06-04） 之前的行为相同。
SQLITE_DBCONFIG_DQS_DML 此选项仅激活或停用 DML 语句（即 DELETE、INSERT、SELECT 和 UPDATE 语句）的旧版双引号字符串文字错误功能。
SQLITE_DBCONFIG_DQS_DDL 此选项可激活或停用 DDL 语句（如 CREATE TABLE 和 CREATE INDEX）的旧版双引号字符串文字错误功能。
SQLITE_DBCONFIG_ENABLE_VIEW 此选项用于启用或禁用 CREATE VIEW。
SQLITE_DBCONFIG_LEGACY_FILE_FORMAT 此选项可激活或停用旧版文件格式标志。
SQLITE_DBCONFIG_TRUSTED_SCHEMA 此选项告诉 SQLite 假设数据库架构（即 sqlite_master 表的内容）不受恶意内容的污染。禁用受信任架构选项后，SQLite 会采取额外的防御措施来保护应用程序免受损害，包括：<ul><li> Prohibit the use of SQL functions inside triggers, views, CHECK constraints, DEFAULT clauses, expression indexes, partial indexes, or generated columns unless those functions are tagged with SQLITE_INNOCUOUS. </li><li> Prohibit the use of virtual tables inside of triggers or views unless those virtual tables are tagged with SQLITE_VTAB_INNOCUOUS. </ul> This setting defaults to "on" for legacy compatibility, however all applications are advised to turn it off if possible. This setting can also be controlled using the PRAGMA trusted_schema statement.
SQLITE_DBCONFIG_STMT_SCANSTATUS 此选项仅在 SQLITE_ENABLE_STMT_SCANSTATUS 版本中有用。在这种情况下，它会设置或清除一个标志，该标志允许收集 sqlite3_stmt_scanstatus_v2（） 统计信息。对于要收集的统计信息，必须在准备 SQL 语句和步进 SQL 语句时在数据库句柄上设置标志。默认情况下，该标志已设置（启用统计信息收集）。
SQLITE_DBCONFIG_REVERSE_SCANORDER SQLITE_DBCONFIG_REVERSE_SCANORDER 选项更改扫描表和索引的默认顺序，以便扫描从末尾开始并朝着开头工作，而不是从开头开始并朝向结尾工作。设置 SQLITE_DBCONFIG_REVERSE_SCANORDER 与设置 [PRAGMA reverse_unordered_selects] 相同。此配置选项对于应用程序测试非常有用。
