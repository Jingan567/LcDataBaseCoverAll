SQLite 理解的 SQL（某种）

类型

sql 语句 ：：= TYPES [数据类型名称][，数据类型名称][，数据类型 姓名][，...] ;select-stmt
select-stmt ：：= 请参阅 SELECT

在 SELECT 语句之前使用 TYPES 关键字以提供 SQLite ADO.NET 提供程序 后续 SELECT 语句中预期的 return 数据类型列表。

这是专门用于 ADO.NET 数据的 SQLite 的语言扩展（又名 hack） 供应商。它是一个伪语句，这意味着只有 ADO.NET 提供商才能理解 它。

背景

由于 SQLite 的无类型性质，对于某些类型的查询， ADO.NET 提供程序无法确定正确的 return 数据类型。Scalar 和 聚合函数会带来一个特定问题，因为 给定的标量或聚合函数不需要返回任何特定的 数据类型。事实上，标量函数理论上可以返回 查询中的每一行或每列都有不同的数据类型，这是完全合法的 从 SQLite 的角度来看。

由于 ADO.NET 是围绕类型化系统设计的，因此我们将 SQLite 硬塞进 it，则此关键字在无法轻松确定返回类型的情况下帮助提供程序。

此命令必须与 SELECT 语句结合使用。它只是 当 TYPES 关键字及其值与 SELECT 一起传递时有效 语句作为单个分号分隔的单元。

例子

类型 [bigint]， [int]， [mediumint]， [smallint]， [tinyint];
选择 1、2、3、4;

上述查询将返回类型 System.Int64、System.Int32、System.Int32 和 System.Int16 的列 和 System.Byte 分别。

类型 [bigint]， [int]， ， [tinyint];
选择 1、2、3、4;

在此示例中，只有第 1、2 和 4 列具有显式类型化。列 3 的数据类型将通过系统并被正常发现。

类型 真实;
SELECT SUM（Cost） FROM [产品];

上述查询显式告知提供程序 SUM 聚合函数返回 一个 System.Double。

使用说明

  • 不能在 TYPES 语句中使用参数。
  • TYPES 语句后面必须紧跟 SELECT 语句。
  • 在多语句中传递多个 TYPES 和 SELECT 语句是合法的 命令。
  • 如果你愿意，你可以将数据类型括在引号 “” 或方括号 [] 或那些 '' 东西中。
