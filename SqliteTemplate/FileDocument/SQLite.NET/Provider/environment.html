环境变量
SQLite.NET 类库文档
See Also Example

可以识别以下特殊替换令牌，并且 替换为其关联的运行时值，具体取决于上下文， 当本机库预加载读取设置值时 子系统。除非另有明确说明，否则所有这些令牌 区分大小写，并且百分比字符必须为 included 以便他们得到认可。

所有对特殊替换令牌的引用都将扩展为 它们在返回的 Settings 值中的相应值，除非 “No_Tokens”或“No_Tokens_<名称>” 环境变量设置为 [to anything]。

名字 描述
%PreLoadSQLite_AssemblyDirectory% 如果此令牌（区分大小写，并且必须包含 百分比字符）存在于 returned 时，它将被替换为 包含 System.Data.SQLite 程序集的目录。如果 目录不可用，则不会替换令牌。
%PreLoadSQLite_TargetFramework% 如果此令牌（区分大小写，并且必须包含 百分比字符）存在于 返回，则它将被替换为 target 的缩写 framework 属性值。如果 Target Framework 属性值不可用，则令牌将 而不是被替换。
%PreLoadSQLite_XmlConfigDirectory% 如果此令牌（区分大小写，并且必须包含 百分比字符）存在于设置值中 时，它将被替换为 目录。如果 目录不可用，则不会替换令牌。 通常，此令牌只能在 XML 配置中使用 文件本身。

这些环境变量用于控制 System.Data.SQLite 库。所有这些环境变量 是可选的。如果不存在特定的环境变量， XML 配置文件 “System.Data.SQLite.dll.config” 在包含当前正在执行的程序集的目录中（即 包含 System.Data.SQLite 的所有托管组件的 URL） 也将咨询。如果存在，则 XML 配置文件应 结构如下：

        <?xml version="1.0"?>
        <configuration>
          <appSettings>
            <add key="A_Setting" value="String Value" />
            <add key="Another_Setting" value="%ENV_VALUE%" />
          </appSettings>
        </configuration>

对现有环境变量的所有引用都将扩展为 它们在返回的 Settings 值中的相应值，除非 “No_Expand”或“No_Expand_<名称>” 环境变量设置为 [to anything]。

Compact 不支持这些环境变量 由于施加的限制，System.Data.SQLite 的框架版本 通过平台本身;但是，提到的 XML 配置文件 可以改用 above。

名字 描述
AppendManifestToken_SQLiteProviderManifest 如果此环境变量设置为 [为任何内容]，它将是 由 System.Data.SQLite.Linq.SQLiteProviderManifest 类使用 （以及 System.Data.SQLite.EF6.SQLiteProviderManifest 类）添加到 修改将来的提供程序清单令牌，方法是将 现有提供程序清单令牌的环境变量， 如果有的话。通常，为了将构建的提供程序清单 token 的语法正确，环境变量值 [to be appended] 必须以分号开头。
DefaultFlags_SQLiteConnection 如果此环境变量设置为 [为任何内容]，它将是 由 System.Data.SQLite.SQLiteConnection 类用作默认值 所有打开连接的标志（即当它们不存在于 连接字符串）。
Disable_SQLiteLog 如果此配置变量设置为 [任意值]，则 SQLite 日志记录子系统在初始化时最初将被禁用。
Force_SQLiteLog 如果此环境变量设置为 [任意值]，则 SQLite 可以在非默认应用程序中初始化日志记录子系统 域。默认情况下，不允许这样做，因为可能存在 应用程序域卸载问题。
Initialize_SQLiteLog 如果此环境变量设置为 [任意值]，则 SQLite logging 子系统将尝试执行初始化，即使 之前曾尝试过。
No_PreLoadSQLite 如果此环境变量设置为 [to anything]，则本机 库预加载代码将被禁用。默认情况下，本机 库预加载代码将尝试加载原生 SQLite 来自特定于架构的库（例如 “x86”， “amd64”、“x64”）或特定于平台的（例如 “Win32”） 目录 base 目录。
No_SQLiteConnectionNewParser 如果此环境变量设置为 [to anything]，则新的 不会使用连接字符串解析算法。这 环境变量仅用于旧代码。
No_SQLiteFunctions 如果此环境变量设置为 [为任何内容]，则初始 在所有标有 将跳过 SQLiteFunction 属性。通常，此搜索是 每个应用程序域仅由 Static 构造函数执行一次 的 SQLiteFunction 类;但是，这些实现细节 可能会发生变化。
No_SQLiteGetSettingValue 如果此环境变量设置为 [to anything]，则对 GetSettingValue 方法将返回默认值。这将 有效防止所有其他设置值产生任何影响， 包括通过其他支持的环境变量指定的变量 或在关联的 XML 配置文件中。
No_SQLiteLog 如果此环境变量设置为 [任意值]，则 SQLite logging 子系统不会由 SQLiteLog 类初始化; 但是，它仍然可以由外部组件（即 System.Data.SQLite 以外的其他内容）。
No_SQLiteXmlConfigFile 如果此环境变量设置为 [to anything]，则调用 GetSettingValue 方法永远不会导致 XML 配置 正在读取的文件;相反，将返回默认值。这 将有效地阻止通过 XML 指定的任何设置值 配置文件不会产生任何影响。
PreLoadSQLite_AllowBaseDirectoryOnly 如果此环境变量设置为 [to anything]，则基 目录本身被认为对预加载本机 SQLite 库。
PreLoadSQLite_BaseDirectory 如果此环境变量设置为 [为任何内容]，它将是 used 而不是本机的 application 基目录 库预加载器。此环境变量可以特别 在 ASP.NET 和其他直接控制的托管环境中很有用 的托管程序集的位置不受控制 的应用程序。
PreLoadSQLite_BreakIntoDebugger 如果此环境变量设置为 [to anything]，则本机 库 pre-loader 子系统将尝试提供交互式 用户：有机会将 Debugger 附加到当前进程。
PreLoadSQLite_LibraryFileNameOnly 如果此环境变量设置为 [为任何内容]，它将是 用作 要预加载的原生 SQLite 库（例如 “sqlite3.dll” 或 “libsqlite3.so.0”）的
PreLoadSQLite_NoSearchForDirectory 如果此环境变量设置为 [to anything]，则本机 库预加载代码将跳过对本机的搜索 库进行预加载。默认情况下，搜索从该位置开始 当前正在执行的程序集（即包含所有 System.Data.SQLite 的托管组件），然后回退到 应用程序域基目录。
PreLoadSQLite_ProcessorArchitecture 如果此环境变量设置为 [为任何内容]，它将是 used 而不是 PROCESSOR_ARCHITECTURE环境变量来帮助构建路径 进行预加载。
PreLoadSQLite_UseAssemblyDirectory 如果此环境变量设置为 [to anything]，则位置 当前正在执行的程序集（即包含所有 System.Data.SQLite 的托管组件）将用作基础 用于查找要预加载的本机库（即，而不是 使用应用程序域基目录）。
PROCESSOR_ARCHITECTURE 这个环境变量通常由作 系统本身，并且应该反映本机处理器架构 当前进程（例如，在 64 位 x64作系统的值应为“x86”）。
SQLite_ForceLogLifecycle 如果此环境变量设置为 [to anything]，则调用 与 Connections 的生命周期及其 将记录关联的类（例如 LINQ、EF6 等）。
SQLite_ForceLogPrepare 如果此环境变量设置为 [to anything]，则对 准备一个 SQL 查询，而不管 关联的连接。
SQLite_ForceLogRetry 如果此环境变量设置为 [to anything]，则所有内部 将记录语句准备和单步执行期间的重试， 无论关联连接的标志如何。
SQLite_GlobalCommandBehaviors 如果此环境变量设置为 [为任何内容]，它将是 如果可能，则作为 CommandBehavior 标志的列表进行处理。它会 如果无法正确处理，则忽略它。更多详情 请参考 CombineBehaviors 方法。
SQLite_LegacyEncryptPage1 如果此环境变量设置为 [to anything]，则 使用（不支持的）旧版加密的数据库文件 基于 CryptoAPI 的 （RC4） 编解码器将被加密。这是默认设置 版本 1.0.112.3 之前的行为;但是，它现在被禁用（通过 default），因为它可能会导致某些 情况 下。请不要使用此环境变量，除非它 对于您的特定用例是绝对必要的。
SQLite_StrongConnectionPool 如果此环境变量设置为 [to anything]，则默认的 Connection Pool 将使用普通对象而不是弱引用。 这应该可以防止任何池化连接对象永远 由垃圾回收器自动清理。
SQLite_TraceCategories 如果此环境变量设置为 [为任何内容]，它将是 如果可能，则作为要启用的跟踪类别列表进行处理。它 如果无法正确处理，则将被忽略。了解更多 详情请参考 ParseTraceCategories 方法。
TypeName_SQLiteProviderServices 如果此环境变量设置为 [为任何内容]，它将是 由 System.Data.SQLite.SQLiteFactory 类用作类型名称 包含 System.Data.Common.DbProviderServices 实现 那应该被使用。
Use_SQLiteConvert_DefaultDbType 如果此环境变量设置为 [为任何内容]，它将是 由 System.Data.SQLite.SQLiteConvert 类用作默认值 DbType 值，当每个连接的值不是 可用。
Use_SQLiteConvert_DefaultTypeName 如果此环境变量设置为 [为任何内容]，它将是 由 System.Data.SQLite.SQLiteConvert 类用作默认值 键入 name （当每个连接的值不是 可用。
