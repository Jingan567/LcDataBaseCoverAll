SQLite.NET 提供程序功能

此 SQLite 提供程序实现了底层 SQLite 的所有功能 数据库引擎。以下是简要摘要：

  • 在 Visual Studio 2008 上从头开始编写，专门用于 ADO.NET，实现了所有 框架中最近引入的基类和功能，包括自动 事务登记。
  • 支持完整和紧凑的 .NET Framework 以及 本机 C/C++ 开发。与原始sqlite3.dll 100% 二进制兼容。
  • 通过针对 官方 SQLite 3.6.1 或更高版本的库。
  • 完整的实体框架支持 （ADO.NET 3.5 SP1）
  • 在 Compact Framework 上，它比 Sql Server Mobile 更快。 SQLite 的安装大小只是 Sql Mobile 的一小部分。它在运行时使用更少的内存， 运行查询的速度更快，并且数据库文件大小也更小。
  • 加密数据库支持。加密的数据库是完全加密的，并且 支持二进制和明文密码类型。
  • Visual Studio 2005/2008/2010/2012/2013/2015 设计时支持。您可以添加 SQLite database 添加到 Servers 列表中，使用 Query Designer 设计查询， 将表格拖放到类型化 DataSet 上，等等。
  • 在 Visual Studio 中进行完整的 SQLite 架构编辑。您可以创建/编辑 表、视图、触发器、索引、检查约束和外键。
  • 作为单个文件可再发行组件提供（Compact Framework 除外）。核心 sqlite3 代码库和 ADO.NET 包装器 组合成一个多模块组件。
  • 也可作为单独的本机程序集和托管程序集使用，也可以选择与静态链接的 Visual C++ 运行时一起使用。
  • 包括用于 Itanium、x64、x86 和 ARM 处理器的二进制文件。
    当前不包括安腾处理器支持。
  • DbProviderFactory 支持。
  • 完全支持 ATTACHED 数据库。在架构中公开为 Catalogs。克隆连接时，所有附加的数据库都是 自动重新连接到新连接。
  • DbConnection.GetSchema（...） 支持包括 MetaDataCollections、 DataSourceInformation、Columns、Tables、Views、ViewColumns、 Catalogs、Indexes、 IndexColumns、ForeignKeys 和 Triggers。
  • 增强的 DbDataReader.GetSchemaTable（） 功能返回目录、命名空间 以及详细的架构信息，即使对于复杂的查询也是如此。
  • 命名和未命名参数。
  • 完整的 UTF-8 和 UTF-16 支持，每个都具有到本机的优化管道 数据库核心。
  • 多个同步 DataReader（但每个命令一个 DataReader）。
  • 完全支持用户定义的标量和聚合函数，封装到 一个易于使用的基类，其中只需要几个覆盖即可 实现新的 SQL 函数。
  • 完全支持用户定义的分页序列，每一点都非常简单 实现为用户定义的函数，并使用相同的基类。
  • 整个引擎和包装器的完整源代码。无版权。 公有土地。100% 免费用于商业和非商业用途。

分发二进制文件 （Desktop）

使用混合模式程序集时，System.Data.SQLite.DLL 文件 包括所有本机代码和托管代码。在这种情况下，这是唯一的 需要与您的 SQLite.NET 应用程序一起重新分发 DLL。 当使用单独的本机程序集和托管程序集时，System.Data.SQLite.DLL 文件包含所有托管代码，SQLite.Interop.DLL 文件包含所有本机代码。 本机代码有 3 种风格：Win32、Itanium 和 x64 （AMD64）。
当前不包括安腾处理器支持。

分发二进制文件 （Compact Framework）

System.Data.SQLite.DLL 和 SQLite.Interop.XXX.DLL 文件 必须部署在 Compact Framework 上。XXX 是 的内部版本号 System.Data.SQLite 库（例如 “119”）。SQLite.Interop.XXX.DLL 文件是针对 ARM 处理器，而 System.Data.SQLite 是完全托管的 Compact 框架程序集。
