DbProviderFactories 和你

ADO.NET 2.0 的一大新特性是将反射用作 以编程方式实例化数据库提供程序的方法。信息 .NET 用于枚举系统中可用的数据提供程序是相对的 简单。它只是在 machine.config 和您自己的 app.config 文件中查找一些 XML 数据，以告诉它什么是提供程序 已安装的程序集以及这些提供程序所在的程序集。

方案 1：版本无关（不使用全局程序集缓存）

此方法允许您将任何新版本的 System.Data.SQLite.DLL 放入 您的应用程序文件夹，无需修改任何代码或重新编译即可使用它。 将以下代码添加到您的 app.config 文件中：

<configuration>
  <system.data>
    <DbProviderFactories>
      <remove invariant="System.Data.SQLite"/>
      <add name="SQLite Data Provider" invariant="System.Data.SQLite"
           description=".NET Framework Data Provider for SQLite"
           type="System.Data.SQLite.SQLiteFactory, System.Data.SQLite" />
    </DbProviderFactories>
  </system.data>
</configuration>

方案 2：版本相关，使用位于同一文件夹中的 DLL 作为应用程序或全局程序集缓存

此方法扩展了上述 XML 以提供版本号和密钥令牌 的 SQLite DLL，以便可以在与应用程序相同的文件夹中找到它 或在 GAC 中查找。此方法的缺点是 DbProviderFactories 将使用此版本信息仅加载指定的版本。这 表示如果更新 DLL，则还必须更新此 XML。

<configuration>
  <system.data>
    <DbProviderFactories>
      <remove invariant="System.Data.SQLite"/>
      <add name="SQLite Data Provider" invariant="System.Data.SQLite"
           description=".NET Framework Data Provider for SQLite"
           type="System.Data.SQLite.SQLiteFactory, System.Data.SQLite,
                 Version=1.0.119.0, Culture=neutral,
                 PublicKeyToken=db937bc2d44ff139"/>
    </DbProviderFactories>
  </system.data>
</configuration>

以下 C# 代码演示 通过 DbProviderFactories 实例化 SQLite：

      DbProviderFactory fact = DbProviderFactories.GetFactory("System.Data.SQLite");
      using (DbConnection cnn = fact.CreateConnection())
      {
        cnn.ConnectionString = "Data Source=test.db3";
        cnn.Open();
      }
