SQLiteChangeSetConflictType 枚举

此枚举类型表示在应用更改集或补丁集的更改时看到的一种冲突类型。

public enum SQLiteChangeSetConflictType

成员

会员名称 描述
Data 如果数据库中存在具有所需 PRIMARY KEY 字段的行，但更新修改的一个或多个其他（非主键）字段不包含预期的“before”值，则在处理 DELETE 或 UPDATE 更改时会看到此值。
NotFound 如果数据库中不存在具有所需 PRIMARY KEY 字段的行，则在处理 DELETE 或 UPDATE 更改时会看到此值。在这种情况下，没有冲突的行。调用 GetConflictValue 方法的结果是不确定的。
Conflict 如果作会导致重复的主键值，则在处理 INSERT 更改时会看到此值。在这种情况下，冲突的行是具有匹配主键的数据库行。
Constraint 如果在应用更改时发生非外键约束冲突（即 UNIQUE、CHECK 或 NOT NULL 约束），则冲突回调将看到此值。在这种情况下，没有冲突的行。调用 GetConflictValue 方法的结果是不确定的。
ForeignKey 如果启用了外键处理，并且应用更改会使数据库处于包含外键冲突的状态，则在提交更改之前，此值将只看到一次。如果冲突处理程序 Omit，则提交更改，包括导致外键约束冲突的更改。或者，如果它返回 Abort，则回滚更改。不提供当前或冲突的行信息。唯一可以对提供的 ISQLiteChangeSetMetadataItem 对象调用的方法是 NumberOfForeignKeyConflicts。
