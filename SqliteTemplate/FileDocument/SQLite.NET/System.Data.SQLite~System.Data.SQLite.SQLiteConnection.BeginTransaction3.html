SQLiteConnection.BeginTransaction（IsolationLevel） 方法

如果连接上尚未激活 SQLiteTransaction，则创建一个新的 SQLiteTransaction。

new public SQLiteTransaction BeginTransaction(
   IsolationLevel isolationLevel
);

Parameters

isolationLevel
支持的隔离级别包括 Serializable、ReadCommitted 和 Unspecified。

返回值

返回 SQLiteTransaction 对象。

言论

Unspecified 将使用连接字符串中指定的默认隔离级别。如果未在连接字符串中指定隔离级别，则使用 Serializable。可序列化事务是默认值。在此模式下，引擎会立即锁定数据库，并且其他线程无法开始事务。其他线程可以从数据库中读取，但不能写入。使用 ReadCommitted 隔离级别，将根据需要延迟和提升锁。多个线程可以在 ReadCommitted 模式下启动事务，但是如果一个线程尝试在另一个线程具有 ReadCommitted 锁时提交事务，则它可能会超时或导致两个线程死锁，直到达到两个线程的 CommandTimeout。
